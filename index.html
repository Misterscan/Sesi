An AI-Native Programming Language

Built on a robust TypeScript tree-walking interpreter, Sesi treats AI as a core primitive. Stop wrestling with string concatenations and SDK wrappers. Write intelligent logic with native prompt blocks, seamless Gemini model calls, and integrated file I/O.

Get Started Explore Features 
prompt extract_data {
    "Extract the key entities from the following text and return as JSON."
}

fn process_data(input_file) {
    try {
        let data = read_file(input_file);
        let result = model("gemini-3.1-pro-preview") {
            extract_data
            data
        };
        write_file("output.json", result);
    } catch (err) {
        write_file("error.log", err);
    }
}

Language Primitives

📝

Native Prompt Blocks

Prompts are first-class citizens. Use the prompt keyword to define reusable, clean instruction blocks without messy string manipulation or external template files.

🧠

First-Class Model Calls

Invoke AI natively with the built-in model() function. Sesi is deeply integrated with Gemini models, allowing you to pass prompts and data directly into the language syntax.

⚡

JSON Structured Output

No more parsing unpredictable text. Sesi natively supports structured JSON output, ensuring your model responses map cleanly to usable data structures in your logic.

🛡️

Integrated Try/Catch

AI can be unpredictable. Sesi includes native try/catch error handling to gracefully manage model hallucinations, unexpected outputs, or file system failures.

📁

Native File System I/O

Interact with your local environment effortlessly. Use built-in read_file() and write_file() functions to ingest data and save AI-generated results directly to disk.

🛠️

Tool Calling & Memory

Extend your AI's capabilities with native tool_call support and built-in memory management, all executed seamlessly by Sesi's underlying TypeScript tree-walking interpreter.
